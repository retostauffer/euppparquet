EUPP Benchmark Dataset API

Gridded Analysis Data

Single date; all hours and parameters:
{{ request.scheme }}://{{ request.META.HTTP_HOST }}/api/analysis/2017-01-01

Singel date; two hours hour, single parameter:
{{ request.scheme }}://{{ request.META.HTTP_HOST }}/api/analysis/2017-01-01?param=q700&time=12:14

Date range, single hour, single parameter:
{{ request.scheme }}://{{ request.META.HTTP_HOST }}/api/analysis/2017-01-01/2017-01-31?param=q700&time=12

Appending list returns the data in 'list style' instead of 'record style' which can be very handy when converting the data to a pandas.DataFrame or data.frame in R.
{{ request.scheme }}://{{ request.META.HTTP_HOST }}/api/analysis/2017-01-01?list

Gridded Forecast Data

Single date; one parameter
{{ request.scheme }}://{{ request.META.HTTP_HOST }}/api/forecast/ens/2017-01-01?param=2t

Single date; single step, single parameter:
{{ request.scheme }}://{{ request.META.HTTP_HOST }}/api/forecast/ens/2017-01-01?step=24&param=2t

Single date; multiple steps and parameters:
{{ request.scheme }}://{{ request.META.HTTP_HOST }}/api/forecast/ens/2017-01-01?step=0:6:12:18&param=2t:cp

Date range, single step, single parameter
{{ request.scheme }}://{{ request.META.HTTP_HOST }}/api/forecast/ens/2017-01-01/2017-01-10?param=2t&step=24

Date range, multiple steps, control run (only ensembe member 0)
{{ request.scheme }}://{{ request.META.HTTP_HOST }}/api/forecast/ens/2017-01-01/2017-01-10?step=24:48:72:96&number=0

Same as above but loading efi (extreme forecast index) meta information and not ens (ensemble forecasts)
{{ request.scheme }}://{{ request.META.HTTP_HOST }}/api/forecast/efi/2017-01-01/2017-01-10?step=24:48:72:96&number=0

Same as above but loading hr (high resolution forecast) meta information:
{{ request.scheme }}://{{ request.META.HTTP_HOST }}/api/forecast/hr/2017-01-01/2017-01-10?step=24:48:72:96&number=0

Loading reforecast meta information. Note that only one product ens is available for reforecasts/hindcasts and these forecasts are only available for Mondays and Thursdays. The date/date range defines the reforecast initialization date, the API returns all connected reforecasts (reforecasts over the previous 20 years).
{{ request.scheme }}://{{ request.META.HTTP_HOST }}/api/reforecast/ens/2017-01-02?step=24&param=100u:100v
